===Octeon Linux setting===
1.Insert the Octeon shared memory kernel module:
  "insmod shm-channel.ko"

2.Using the hexdump tool to check if the Kernel to MCP channel in the shared memory establish:
  "hexdump -C -s 0x70040000 -n 0x1000 /dev/mem"

  If everything worked fine, then the "ktom" magic string should be deployed to the share memory as following result:
    70040000  6b 74 6f 6d 13 00 00 00  00 00 00 00 00 00 00 00  |ktom............|
    70040010  00 00 00 00 00 00 00 00  00 00 00 00 00 00 00 00  |................|
    *
    70041000

===Octeon Linux userspace tool===
1.Userspace kernel to MCP channel access utility usage:
  "channel-util -h"
    -h                                               Help message.
    -c                                               Reset the channel.
    -d [type]                                        Remove the given type data from the channel.
    -r [type] [data0] [data1] [data2] ...            Insert the given type data to the channel.
    -b [index] [data]                                Insert the byte data to the channel.
    -s [index] [string]                              Insert the string to the channel.
    -n [oem id] [channel] [command type] [data0] ... Insert the NCSI response record.
    -w [oem id] [channel] [command type]             Wipe the NCSI response record.
    [type]                                           Fields Type 0x00 - 0xff
    [index]                                          Type Index 0x00 - 0x1f
    [oem id]                                         NCSI OEM ID.
       0                                             - Standard NCSI
       1                                             - DELL NCSI (Manufacturer ID 0x02A2)

Note: The MCP FW will looking for the NCSI record only when the request NCSI command needs the Kernel to fill in the response data.
Besides, the MCP FW will based on the request NCIS command is package-level or channel level to determine the searched NCSI record channel ID.
The channel ID of the package-level NCSI command record must be 0x1f, and the channel-level NCSI command record is channel specifics.

2.POST standard NCSI record usage:
  channel-util -n 0 <channel ID> <payload>
  e.g., "channel-util -n 0 0x01 0x95 0xf1 0xf1 0xf1 0x00 0x00 0x00 0x00 0x00 0x4f 0x43 0x54 0x45 0x4f 0x4e 0x20 0x42 /
      0x45 0x54 0x41 0x00 0x01 0x01 0x00 0xa0 0xaa 0xaa 0xbb 0xbb 0xcc 0xcc 0xdd 0xdd 0x00 0x00 0x02 0xa2"
    <OEM ID = 0> <channel id = 1> <command typ = 0x95> <NCSI version = 1.1.1.0> <reserved = 0 0 0 Alpha2 = 0> /
    <Firmware name string = OCTEON BETA> <Firmware version = 01.01.00.a0> <PCI DID = 0xaaaa> <PCI VID = 0xbbbb> /
    <PCI SSID = 0xcccc> <PCI SVID = 0xdddd> <Manufacturer ID = 0x000002a2>
    post Get Version ID response (0x95) channel 0 record.

Note: The standard NCSI command record only required the NCSI response command payload.
(NCSI control packet header and the checksum will be responsible by the MCP FW)

3.POST DELL OEM NCSI record usage:
  channel-util -n 1 <channel ID> <payload>
  e.g., "channel-util -n 1 0x1f 0x1a 0x02 0x1a 0x00 0x04"
    <OEM ID = 1> <channel id = 0x1f> <command id = 0x1a> <payload version = 0x02> <command id = 0x1a> /
    <Supported Versions = 4>
    post Get Supported Payload Version Response (0x1A) channel 0x1f record.

Note: The OEM NCSI command record only required the NCSI command payload but without the DELL Manufacture ID.
(NCSI control packet header and the checksum will be responsible by the MCP FW)

===OpenBMC MCTP over SMBUS tool===
1.openbmc mctp-util usage:
  "mctp-util -h"
    Usage: mctp-util [options] <bus#> <dst_eid> <type> <cmd payload>
    Sends MCTP data over SMbus
    Options
           -h             this help
           -d             decode response
           -s <addr>      slave address (Hex). Default: 0x32
           -d             decode response
    Command fields
           <bus#>         I2C Bus number
           <dst_eid>      destination EID
           <type>         MCTP message type
               0            - MCTP Control Message
               1            - PLDM
               2            - NC-SI
               3            - Ethernet
               4            - NVME
               5            - SPDM
               6            - Secured Messages

2.NCSI over MCTP over SMBUS command format:
  mctp-util -s <addr> <bus> <dst_eid> <type> <packet header(16 bytes)> ... /
  <payload(optional)> <payload pad(optional)> <checksum(31:24)> <checksum(23:16)>
  <checksum(15:8)> <checksum(7:0)>

3.NCSI packet header formats:
  <mc_id> <hdr_rev> <reserved> <iid> <cmd> <channel_id> <pay_len(12:8)> <pay_len(7:0)> /
  <reserved(63:56)> <reserved(55:48)> <reserved(47:40)> <reserved(39:32)> /
  <reserved(31:24)> <reserved(23:16)> <reserved(15:8)> <reserved(7:0)>
